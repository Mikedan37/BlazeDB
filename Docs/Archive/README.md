# Documentation archive

This directory contains **historical and superseded** documentation: release milestone notes, older design write-ups, and material kept for context. It is **not** the canonical source for current BlazeDB OSS behavior.

**How to use this tree**

- Prefer [Docs/README.md](../README.md) and **Canonical docs** listed there for APIs, CI, and product guarantees.
- Treat claims here (performance superlatives, sync completeness, timelines) as **time-stamped context**, not promises about the current default `Package.swift` product unless a maintained doc explicitly points here.

Individual files are not rewritten in bulk; when in doubt, verify against the root `README.md`, `Docs/Testing/CI_AND_TEST_TIERS.md`, and `Docs/Status/DISTRIBUTED_TRANSPORT_DEFERRED.md`.
